Key Differences

  • Before: Fixed maxWidth of 320px prevents preview from using available height - appears small regardless of modal size
  • After: Height-based scaling with h-full class - preview fills available vertical space and scales proportionally
  • Aspect Ratio: Both maintain 8.5:11 ratio, but the AFTER version uses space more efficiently
  • Responsive: Try resizing with the slider - AFTER adapts to available space, BEFORE stays fixed

CSS Changes

maxWidth: '320px',     // Removed - prevented scaling
height: '100%',         // Added via h-full class
maxWidth: '100%',      // Added - prevents overflow
aspectRatio: '8.5 / 11', // Kept - maintains paper ratio

Benefits

  • Preview is more readable on large screens
  • Better use of available vertical space in modal
  • Still maintains correct 8.5:11 aspect ratio
  • Scales down gracefully on smaller screens
  • No horizontal overflow or clipping
